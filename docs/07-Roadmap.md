# 1. Objetivo

O Roadmap representa o plano oficial de evolução profissional do usuário.

Seu propósito é organizar todas as Competências em uma sequência lógica de aprendizado.

Nenhuma missão deverá existir sem estar vinculada a uma etapa do Roadmap.

---

# 2. Estrutura

Cada etapa deverá possuir:

- Nome
- Objetivo
- Competências
- Tecnologias
- Projetos
- Missões
- Critério de conclusão

---

# 3. Implementação

Criar uma Database chamada **Roadmap**.

Adicionar as seguintes propriedades:

| Propriedade | Tipo |
|-------------|------|
| Etapa | Title |
| Ordem | Number |
| Objetivo | Text |
| Status | Status |
| Data Início | Date |
| Data Fim | Date |
| Competências | Relation |
| Missões | Relation |
