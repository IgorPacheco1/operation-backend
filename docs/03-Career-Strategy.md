# 1. Objetivo

Este documento define toda a estratégia de evolução profissional do Operation Backend.

Seu propósito é responder uma única pergunta:

> **Como transformar um estudante em um candidato competitivo para conquistar um estágio em Desenvolvimento Backend no Banco Inter?**

Todas as decisões arquiteturais do projeto deverão estar alinhadas com esta estratégia.

---

# 2. Visão Estratégica

O Operation Backend não foi concebido apenas como um organizador de estudos.

Ele foi projetado para funcionar como um sistema operacional de evolução profissional.

Seu papel é eliminar a necessidade de decidir diariamente o que estudar.

O sistema deverá transformar um objetivo profissional em uma sequência lógica de competências, tecnologias, projetos e evidências.

Todo aprendizado deverá possuir um propósito claro.

Toda competência deverá produzir um resultado mensurável.

Toda evolução deverá ser registrada.

---

# 3. Objetivo Final

## Objetivo Principal

Conquistar uma vaga de Estágio em Desenvolvimento Backend no Banco Inter.

Para isso, o usuário deverá desenvolver competências técnicas, construir projetos relevantes, fortalecer sua presença profissional e produzir evidências capazes de demonstrar sua evolução.

---

# 4. Perfil Profissional Alvo

Ao final da jornada o usuário deverá ser capaz de:

- Desenvolver APIs REST completas.
- Modelar bancos de dados relacionais.
- Construir aplicações seguindo boas práticas.
- Trabalhar com Git e GitHub.
- Desenvolver testes automatizados.
- Implementar autenticação utilizando JWT.
- Containerizar aplicações.
- Realizar deploy.
- Trabalhar utilizando metodologias ágeis.
- Aprender novas tecnologias de forma independente.

Mais importante do que conhecer muitas ferramentas é demonstrar capacidade de construir soluções completas.

---

# 5. Filosofia da Estratégia

Toda decisão do sistema seguirá cinco princípios.

## Construir antes de consumir

O aprendizado deverá acontecer através da construção de software.

A teoria existe para apoiar a prática.

---

## Aprender com propósito

Nenhuma tecnologia será estudada sem um objetivo claro.

Todo conteúdo deverá contribuir para o desenvolvimento de uma competência.

---

## Evolução contínua

O progresso diário é mais importante do que grandes períodos isolados de estudo.

---

## Evidências acima de conhecimento

Conhecimento sem aplicação possui pouco valor profissional.

Projetos publicados possuem maior impacto do que listas de tecnologias.

---

## Foco no objetivo

Toda decisão deverá aproximar o usuário do objetivo final.

# 8. Jornada de Evolução

Esta é a ordem oficial de desenvolvimento das competências.

Cada etapa prepara o usuário para a próxima.

## Etapa 1 — Fundamentos

Objetivo:

Construir uma base sólida na linguagem Java e no ecossistema de desenvolvimento.

Competências:

- Programação Orientada a Objetos
- Git
- GitHub
- Maven
- SQL

Resultado esperado:

Capacidade de desenvolver aplicações Java organizadas e versionadas.

---

## Etapa 2 — Backend

Objetivo:

Construir APIs REST completas.

Competências:

- Spring Boot
- Controllers
- Services
- DTO
- Exception Handler
- Persistência de Dados

Resultado esperado:

Primeira versão funcional do Easy Finance.

---

## Etapa 3 — Segurança

Objetivo:

Adicionar autenticação profissional.

Competências:

- Spring Security
- JWT
- BCrypt

Resultado esperado:

Sistema completo de login e autorização.

---

## Etapa 4 — Qualidade

Objetivo:

Garantir confiabilidade da aplicação.

Competências:

- JUnit
- Mockito
- Boas práticas
- SOLID

Resultado esperado:

Projeto com testes automatizados.

---

## Etapa 5 — DevOps

Objetivo:

Preparar o software para produção.

Competências:

- Docker
- Docker Compose
- Deploy
- GitHub Actions

Resultado esperado:

Aplicação publicada online.

---

## Etapa 6 — Cloud

Objetivo:

Compreender os fundamentos de infraestrutura moderna.

Competências:

- AWS
- Conceitos de Cloud
- Serviços básicos

Resultado esperado:

Capacidade de realizar deploy em ambientes Cloud.
